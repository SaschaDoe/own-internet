Welcome to Digital Independence

What Does "Build Your Own Internet" Mean?

Imagine if Facebook, Google, Netflix, and WhatsApp all disappeared tomorrow. What would you do? This guide teaches you how to create your own versions of these services that YOU control. No more:

  • Companies selling your data
  • Services shutting down unexpectedly
  • Censorship and content removal
  • Monthly subscription fees
  • Internet outages affecting your work

📖 What is Self-Hosting?

Self-hosting means running software on your own computer or server instead of using someone else's. Think of it like this:

  • Normal way: You upload photos to Google Photos
  • Self-hosted: You run your own photo app on your computer

🖥️ What is a Server?

A server is just a computer that's always on and connected to the internet. It can be:

  • An old laptop in your closet
  • A tiny Raspberry Pi ($35 computer)
  • A rented computer in a data center (VPS)

🐳 What is Docker?

Docker is like an app store for servers. Instead of complex installation:

  • Download a "container" (pre-packaged app)
  • Run one command
  • Your service is ready!

These platforms install multiple services with just a few clicks!

Option 1: CasaOS (Recommended for Beginners)

# Works on any Debian/Ubuntu system or Raspberry Pi curl -fsSL https://get.casaos.io | sudo bash # Access at: http://your-server-ip:81 # Install apps from the app store with one click!
What is CasaOS? It's like having an app store on your server. Browse apps, click install, done! Perfect for beginners with a beautiful interface.

Option 2: Yunohost (More Apps Available)

# Fresh Debian install recommended curl https://install.yunohost.org | bash # Follow the wizard for domain setup # Access at: https://your-domain.com

Option 3: Docker Compose (More Control)

# Install Docker curl -fsSL https://get.docker.com | sh sudo usermod -aG docker $USER # Create docker-compose.yml file # Copy our starter template below
Starter docker-compose.yml for essential services: 
version: '3'
services:
  # Nextcloud - Your personal cloud
  nextcloud:
    image: nextcloud
    ports:
      - 8080:80
    volumes:
      - nextcloud_data:/var/www/html
    environment:
      - MYSQL_HOST=db
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
      - MYSQL_PASSWORD=secure_password

  # Database for Nextcloud
  db:
    image: mariadb
    volumes:
      - db_data:/var/lib/mysql
    environment:
      - MYSQL_ROOT_PASSWORD=very_secure_password
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
      - MYSQL_PASSWORD=secure_password

  # Vaultwarden - Password manager
  vaultwarden:
    image: vaultwarden/server
    ports:
      - 8081:80
    volumes:
      - vaultwarden_data:/data

volumes:
  nextcloud_data:
  db_data:
  vaultwarden_data:
# Save the above as docker-compose.yml, then: docker-compose up -d # Access Nextcloud at: http://your-ip:8080 # Access Vaultwarden at: http://your-ip:8081

Essential First Steps

1. Domain Name (Optional but Recommended)

Get a domain from Namecheap or Porkbun (~$10/year)

Point it to your server IP using DNS settings

Use DuckDNS for a free subdomain if on budget

2. SSL Certificates (Free!)

Use Let's Encrypt for HTTPS

Most platforms handle this automatically

Nginx Proxy Manager makes it super easy

3. Backup Strategy

3-2-1 rule: 3 copies, 2 different media, 1 offsite

Use Duplicati or Borg for automated backups

Test your backups regularly!

⚠️ Security Basics - Don't Skip These!
  • Change ALL default passwords immediately
  • Enable automatic security updates
  • Use a firewall (ufw is simple: sudo ufw allow 22,80,443/tcp && sudo ufw enable)
  • Set up fail2ban to block brute force attacks
  • Never expose services directly - use a reverse proxy

Reverse Proxy with Nginx Proxy Manager

A reverse proxy is like a receptionist for your server. Instead of remembering ports like :8080, :8081, you can use nice URLs like files.yourdomain.com
